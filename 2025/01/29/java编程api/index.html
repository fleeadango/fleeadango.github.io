Stack

常用的 Stack API：

1. push(E item)

  • 功能：将元素推入栈中。
  • 参数：item - 要推入栈中的元素。
  • 返回值：没有返回值。

2. pop()

  • 功能：移除并返回栈顶元素。
  • 返回值：栈顶元素。
  • 异常：如果栈为空，抛出 EmptyStackException。

3. peek()

  • 功能：返回栈顶元素，但不移除它。
  • 返回值：栈顶元素。
  • 异常：如果栈为空，抛出 EmptyStackException。

4. empty()

  • 功能：检查栈是否为空。
  • 返回值：如果栈为空，返回 true，否则返回 false。

5. search(Object o)

  • 功能：查找指定元素的位置（从栈顶开始，位置从 1 开始）。
  • 返回值：如果元素存在，返回其位置（从栈顶开始，位置从 1 开始），否则返回 -1。

6. size() (继承自 Vector 类)

  • 功能：返回栈中元素的数量。
  • 返回值：栈中的元素数量。

7. clear() (继承自 Vector 类)

  • 功能：清空栈中所有的元素。
  • 返回值：没有返回值。

String

常用的 String API：

1. 创建和初始化字符串

  • String(String original)
    构造一个新的字符串对象，该对象的内容是传入的字符串。

  • String(char[] value)
    用字符数组创建一个新的字符串。

2. 字符串基本操作

  • length()
    返回字符串的长度（字符数）。

  • charAt(int index)
    返回指定位置的字符。

  • substring(int start)
    返回从指定索引到字符串末尾的子字符串。

  • substring(int start, int end)
    返回从 start 索引到 end 索引（不包含 end）的子字符串。

  • indexOf(int ch)
    返回指定字符首次出现的位置索引，如果没有找到返回 -1。

  • indexOf(String str)
    返回指定子字符串首次出现的位置索引。

  • lastIndexOf(int ch)
    返回指定字符最后一次出现的位置索引。

3. 字符串的转换

  • toLowerCase()
    将字符串中的所有字符转换为小写。

  • toUpperCase()
    将字符串中的所有字符转换为大写。

  • trim()
    去除字符串两端的空格。

  • valueOf(Object obj)
    将对象转换为字符串。

4. 字符串的匹配与比较

  • equals(Object obj)
    比较字符串内容是否相同。

  • equalsIgnoreCase(String anotherString)
    忽略大小写比较字符串内容。

  • startsWith(String prefix)
    检查字符串是否以指定的前缀开头。

  • endsWith(String suffix)
    检查字符串是否以指定的后缀结尾。

  • matches(String regex)
    检查字符串是否匹配给定的正则表达式。

5. 字符串的连接与替换

  • concat(String str)
    将指定的字符串连接到当前字符串的末尾。

  • replace(char oldChar, char newChar)
    替换字符串中的指定字符。

  • replaceAll(String regex, String replacement)
    使用正则表达式替换匹配的部分。

  • split(String regex)
    根据指定的正则表达式分割字符串。

  • join(CharSequence delimiter, CharSequence... elements)
    使用指定的分隔符连接多个字符串元素。

6. 其他常用方法

  • contains(CharSequence sequence)
    检查字符串是否包含指定的子字符串。

  • isEmpty()
    检查字符串是否为空（长度为 0）。

  • toCharArray()
    将字符串转换为字符数组。

常见转换方法

1. 字符串转为数字

  • Integer.parseInt(String s)
    将字符串转换为 int 类型。若字符串无法解析为有效的整数，将抛出 NumberFormatException。

  • Double.parseDouble(String s)
    将字符串转换为 double 类型。若字符串无法解析为有效的浮点数，将抛出 NumberFormatException。

  • Float.parseFloat(String s)
    将字符串转换为 float 类型。若字符串无法解析为有效的浮点数，将抛出 NumberFormatException。

  • Long.parseLong(String s)
    将字符串转换为 long 类型。若字符串无法解析为有效的长整型数值，将抛出 NumberFormatException。

  • Short.parseShort(String s)
    将字符串转换为 short 类型。

  • Byte.parseByte(String s)
    将字符串转换为 byte 类型。

  • Character.getNumericValue(char ch)
    将字符转换为数字。适用于单个字符的转换。

2. 数字转为字符串

  • String.valueOf(int i)
    将 int 类型转换为字符串。

  • String.valueOf(double d)
    将 double 类型转换为字符串。

  • String.valueOf(Object obj)
    将对象转换为字符串。如果对象为 null，返回字符串 "null"。

  • Integer.toString(int i)
    将 int 类型转换为字符串。

  • Double.toString(double d)
    将 double 类型转换为字符串。

3. 自动拆装箱与手动拆装箱

  • 自动拆装箱 (Autoboxing)
    Java 会自动将基本类型转换为包装类型。例如，int 转为 Integer，double 转为 Double 等。

  • 手动拆装箱 (Unboxing)
    手动将包装类转换为基本类型。

4. 基本数据类型之间的转换

  • int 转为 long

  • long 转为 int 强制类型转换（long 可能会丢失精度）。

  • int 转为 double

  • double 转为 int 强制类型转换（double 小数部分会被丢弃）。

  • float 转为 double

  • double 转为 float 强制类型转换（double 可能会丢失精度）。

5. 其他常见转换

  • Boolean.parseBoolean(String s)
    将字符串转换为布尔值。如果字符串为 "true"（不区分大小写），返回 true，否则返回 false。

  • Boolean.valueOf(String s)
    与 Boolean.parseBoolean() 相似，但它返回 Boolean 对象。

  • String.format()
    格式化数字和其他数据类型为字符串。

HashMap

HashMap 是 Java 中最常用的集合类之一，它实现了 Map 接口，用于存储键值对（key-value）。下面是一些常见的 HashMap API 方法：

1. 基本操作

  • put(K key, V value)
    将指定的键值对添加到 HashMap 中。如果键已存在，则更新其对应的值。

  • get(Object key)
    根据指定的键获取对应的值。如果键不存在，返回 null。

  • remove(Object key)
    删除指定键的键值对。如果存在，则返回被删除的值；如果键不存在，返回 null。

  • containsKey(Object key)
    检查 HashMap 是否包含指定的键。

  • containsValue(Object value)
    检查 HashMap 是否包含指定的值。

  • isEmpty()
    检查 HashMap 是否为空。

  • size()
    获取 HashMap 中键值对的数量。

  • clear()
    清空 HashMap 中的所有键值对。

2. 获取视图

  • keySet()
    返回一个包含 HashMap 中所有键的 Set 视图。

  • values()
    返回一个包含 HashMap 中所有值的 Collection 视图。

  • entrySet()
    返回一个包含 HashMap 中所有键值对（Map.Entry 对象）的 Set 视图。

3. 其他辅助方法

  • putIfAbsent(K key, V value)
    如果指定的键不存在，则将键值对添加到 HashMap 中。如果键已存在，不做任何操作。

  • computeIfAbsent(K key, Function<? super K, ? extends V> mappingFunction)
    如果指定的键不存在，则使用提供的函数计算并插入值。

  • replace(K key, V oldValue, V newValue)
    如果 HashMap 中的键存在且其值与 oldValue 相等，则替换为 newValue。

  • replace(K key, V value)
    如果 HashMap 中的键存在，则替换其对应的值。

  • forEach(BiConsumer<? super K, ? super V> action)
    对 HashMap 中的每个键值对执行指定的操作。

  • getOrDefault(Object key, V defaultValue)
    如果指定的键存在，返回其对应的值；否则返回默认值。

  • remove(Object key, Object value)
    如果指定的键存在并且其对应的值与 value 相等，则移除该键值对。

  • replaceAll(BiFunction<? super K, ? super V, ? extends V> function)
    使用指定的函数替换 HashMap 中每个键对应的值。

4. 线程安全相关

虽然 HashMap 本身不是线程安全的，但可以通过一些方法保证线程安全：

5. 重要注意事项

  • HashMap 中的键是 无序 的，插入顺序不被保证。如果需要按插入顺序维护键值对，可以使用 LinkedHashMap。
  • HashMap 允许 null 键和 null 值，但其性能可能受影响。
  • HashMap 的容量和负载因子可以通过构造函数进行设置，调整它们有助于优化性能。

这些是 HashMap 中常见的 API 方法，它们帮助你有效地操作和管理键值对数据。

HashSet

HashSet 是 Java 中基于哈希表实现的集合类，它实现了 Set 接口，具有元素唯一性。以下是 HashSet 中常用的 API 方法：

1. 基本操作

  • add(E e)
    向 HashSet 中添加元素，如果元素已存在，则不添加，并返回 false；否则，返回 true。

  • remove(Object o)
    移除 HashSet 中指定的元素。如果移除成功，返回 true；否则返回 false。

  • contains(Object o)
    检查 HashSet 是否包含指定元素。

  • size()
    返回 HashSet 中元素的个数。

  • isEmpty()
    检查 HashSet 是否为空。

  • clear()
    清空 HashSet 中的所有元素。

2. 批量操作

  • addAll(Collection<? extends E> c)
    将指定集合中的所有元素添加到 HashSet 中。

  • removeAll(Collection<?> c)
    移除 HashSet 中与指定集合中元素相同的所有元素。

  • retainAll(Collection<?> c)
    保留 HashSet 中与指定集合中相同的元素，其他元素删除。

  • containsAll(Collection<?> c)
    检查 HashSet 是否包含指定集合中的所有元素。

3. 迭代操作

  • iterator()
    返回一个迭代器，用于遍历 HashSet 中的元素。

  • forEach(Consumer<? super E> action)
    对 HashSet 中的每个元素执行指定操作。

4. 集合视图

  • toArray()
    将 HashSet 转换为包含所有元素的数组（返回 Object[] 类型）。

  • toArray(T[] a)
    将 HashSet 转换为指定类型的数组。

5. 其他辅助方法

  • clone()
    创建并返回 HashSet 的一个副本。

  • isSubset(Collection<?> c)
    检查当前集合是否是另一个集合的子集。该方法通过 retainAll 来实现。

  • hashCode()
    返回 HashSet 的哈希码，通常用于与其他集合进行比较或存储。

  • equals(Object o)
    检查 HashSet 是否与另一个对象相等。两个 HashSet 相等的条件是它们包含相同的元素。

6. 线程安全

HashSet 本身是 非线程安全 的。如果需要在多线程环境下使用，可以使用以下方法使其线程安全：

  • Collections.synchronizedSet(Set<T> s)

    将

    包装为线程安全的集合。

7. 重要注意事项

  • HashSet 是 无序 的，不保证元素的顺序。
  • 它的元素是 唯一 的，不能包含重复的元素。
  • HashSet 允许一个 null 元素，但只有一个。
  • HashSet 的元素根据其 哈希值 存储和检索，因此其查找、删除操作的时间复杂度为 O(1)。

以上是 HashSet 的常见 API 方法，这些方法使得你可以灵活地操作集合中的元素，并提供了高效的性能。

ArrayList

ArrayList 是 Java 中最常用的动态数组实现，它属于 List 接口。以下是 ArrayList 中常见的 API 方法：

1. 基本操作

  • add(E e)
    向 ArrayList 的末尾添加元素。

  • add(int index, E element)
    在指定位置插入元素，其他元素会自动向后移动。

  • remove(Object o)
    移除第一次出现的指定元素，如果元素存在则返回 true。

  • remove(int index)
    移除指定索引位置的元素，返回被移除的元素。

  • get(int index)
    返回指定索引位置的元素。

  • set(int index, E element)
    替换指定索引位置的元素，返回旧元素。

  • clear()
    清空 ArrayList 中的所有元素。

  • contains(Object o)
    检查 ArrayList 是否包含指定元素。

  • size()
    返回 ArrayList 中的元素个数。

  • isEmpty()
    检查 ArrayList 是否为空。

  • indexOf(Object o)
    返回指定元素第一次出现的索引位置，如果没有找到则返回 -1。

  • lastIndexOf(Object o)
    返回指定元素最后一次出现的索引位置，如果没有找到则返回 -1。

2. 批量操作

  • addAll(Collection<? extends E> c)
    将指定集合中的所有元素添加到 ArrayList 的末尾。

  • addAll(int index, Collection<? extends E> c)
    从指定索引位置开始将指定集合中的所有元素添加到 ArrayList。

  • removeAll(Collection<?> c)
    移除 ArrayList 中与指定集合中元素相同的所有元素。

  • retainAll(Collection<?> c)
    保留 ArrayList 中与指定集合中相同的元素，其他元素删除。

  • containsAll(Collection<?> c)
    检查 ArrayList 是否包含指定集合中的所有元素。

  • toArray()
    将 ArrayList 转换为一个包含所有元素的数组。

  • toArray(T[] a)
    将 ArrayList 转换为指定类型的数组。

3. 迭代操作

  • iterator()
    返回一个迭代器，用于遍历 ArrayList 中的元素。

  • listIterator()
    返回一个 ListIterator，用于双向遍历 ArrayList。

  • forEach(Consumer<? super E> action)
    对 ArrayList 中的每个元素执行指定操作。

4. 其他辅助方法

  • subList(int fromIndex, int toIndex)
    返回 ArrayList 中从 fromIndex 到 toIndex 之间的子列表（不包含 toIndex）。

  • ensureCapacity(int minCapacity)
    确保 ArrayList 至少能容纳指定数量的元素。

  • trimToSize()
    将 ArrayList 的容量调整为当前元素的大小，释放未使用的内存。

5. 线程安全

ArrayList 本身是 非线程安全 的。如果需要在多线程环境下使用，可以使用以下方法使其线程安全：

  • Collections.synchronizedList(List<T> list)

    将

    包装为线程安全的列表。

6. 重要注意事项

  • ArrayList 是 有序 的，保持元素的插入顺序。
  • 它的元素是 可重复 的，可以包含重复的元素。
  • 由于其实现是基于动态数组，ArrayList 的索引访问非常高效，时间复杂度是 O(1)。
  • 对于大规模数据的插入和删除操作，特别是在列表开头或中间，性能较差，时间复杂度为 O(n)。

以上是 ArrayList 的一些常见 API 方法，它们帮助你管理动态数组并提供了丰富的功能来操作集合中的元素。

Deque

Deque（双端队列）是 Java 集合框架中的一个接口，继承自 Queue 接口，允许从两端插入和删除元素。它提供了比 Queue 更多的操作方法，适用于需要从队列两端进行插入和删除的场景。

Deque 的实现通常是 ArrayDeque 或 LinkedList。

Deque 常用的 API

1. 添加元素的方法

  • void addFirst(E e)
    在队列的前端插入元素 e。如果队列已满，抛出 IllegalStateException。
  • void addLast(E e)
    在队列的末端插入元素 e。如果队列已满，抛出 IllegalStateException。
  • boolean offerFirst(E e)
    在队列的前端插入元素 e。如果队列满了，返回 false，而不是抛出异常。
  • boolean offerLast(E e)
    在队列的末端插入元素 e。如果队列满了，返回 false，而不是抛出异常。

2. 移除元素的方法

  • E removeFirst()
    移除并返回队列的第一个元素。队列为空时，抛出 NoSuchElementException。
  • E removeLast()
    移除并返回队列的最后一个元素。队列为空时，抛出 NoSuchElementException。
  • boolean removeFirstOccurrence(Object o)
    移除队列中首次出现的元素 o（从前端开始搜索）。如果找到并移除元素，则返回 true，否则返回 false。
  • boolean removeLastOccurrence(Object o)
    移除队列中最后一次出现的元素 o（从末端开始搜索）。如果找到并移除元素，则返回 true，否则返回 false。

3. 获取元素的方法

  • E getFirst()
    获取队列的第一个元素，但不移除。队列为空时，抛出 NoSuchElementException。
  • E getLast()
    获取队列的最后一个元素，但不移除。队列为空时，抛出 NoSuchElementException。
  • E peekFirst()
    获取队列的第一个元素，但不移除。如果队列为空，返回 null。
  • E peekLast()
    获取队列的最后一个元素，但不移除。如果队列为空，返回 null。

4. 检查元素的方法

  • boolean isEmpty()
    检查队列是否为空。如果队列为空，返回 true，否则返回 false。
  • int size()
    返回队列中的元素数量。

5. 迭代操作

  • Iterator<E> iterator()
    返回队列的迭代器。
  • Iterator<E> descendingIterator()
    返回队列的逆向迭代器（从队尾到队头迭代）。

6. 清空队列

  • void clear()
    清空队列中的所有元素。
